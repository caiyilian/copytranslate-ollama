# CopyTranslator-Ollama 本地翻译助手 — 方案设计

> 基于 CopyTranslator 理念 + 本地 Ollama 专用翻译模型，搭建纯离线、轻量、高效的翻译工作流。

---

## 1. 项目定位

### 1.1 解决的问题

- 科研/工作中频繁阅读外文 PDF/网页，复制后手动粘贴到翻译工具流程繁琐
- PDF 复制内容含多余换行/断句，直接粘贴翻译结果差
- 在线翻译（Google/DeepL）需联网、有隐私风险、可能被墙
- 大模型翻译质量好但通用模型（7B+）跑在本地上太重

### 1.2 核心思路

```
用户复制文本 → 系统监听剪贴板 → 文本净化(去多余换行等)
→ 调用本地 Ollama 翻译模型 → 显示/自动复制/替换选中文本
```

**关键差异 vs CopyTranslator 官方版：**

| 维度 | CopyTranslator 官方 | 本项目 |
|------|-------------------|--------|
| 翻译引擎 | Google/DeepL/百度等云端 API | **本地 Ollama 专用翻译模型** |
| 联网要求 | 必须联网 | **完全离线** |
| 隐私 | 数据经第三方服务器 | **数据不出本机** |
| 速度 | 依赖网络延迟 | **取决于本地 GPU/CPU** |
| 成本 | API 按量付费或有限免费 | **零费用** |
| 模型选择 | 不可控 | **用户自由选择/切换翻译模型** |

## 2. 已就绪的本地翻译模型

当前已下载并验证三个专用翻译模型：

| 模型 | 大小 | 语种 | 速度 | 翻译质量 | 推荐场景 |
|------|------|------|------|---------|---------|
| **translategemma:4b** ★ | 3.3 GB | 55 种 | 中等 | **优秀** — 完整、自然、地道 | 主力翻译，适合质量优先 |
| **hy-mt1.5:1.8b** | 1.1 GB | 33 种 | **最快** | 良好 — 通顺但细节略有冗余 | 快速翻译，适合内存/显存有限 |
| gemma3-translator:1b | 815 MB | 多语种 | 快 | ❌ 只输一半（测试中长句截断） | **不推荐**，保留作兜底 |

**测试对比（英→中）：**

> 原文：The quick brown fox jumps over the lazy dog. Machine translation has made remarkable progress in recent years, enabling seamless communication across languages.

- **translategemma:4b**: "敏捷的棕色狐狸跳过了懒惰的狗。近年来，机器翻译取得了显著进展，使得跨语言交流变得更加顺畅。" ✅
- **hy-mt1.5:1.8b**: "那只快速的棕色狐狸跳过了那只懒狗。近年来，机器翻译取得了显著的进展，使得不同语言之间的交流变得流畅自如。" ✅
- **gemma3-translator:1b**: "敏捷的棕色狐狸跳过懒狗。" ❌（第二句完全丢失）

## 3. 功能清单（按优先级）

### P0 — 核心必做

| # | 功能 | 说明 |
|---|------|------|
| 1 | **剪贴板监听** | 系统托盘运行，实时监听剪贴板变化，检测到新文本自动触发翻译流程 |
| 2 | **文本净化（PDF 断行修复）** | 自动合并被换行/分页断开的段落和句子，针对英文/中文 PDF 优化：<br>- 去除段落内多余换行<br>- 修复断词（如"transla-\ntion" → "translation"）<br>- 合并连字符断词<br>- 保留段落之间的空行分隔 |
| 3 | **Ollama 本地翻译** | 通过 Ollama API (`http://localhost:11434/api/generate`) 调用加载的翻译模型，支持指定源语言/目标语言 |
| 4 | **模型热切换** | 支持在 translategemma:4b / hy-mt1.5:1.8b 之间随时切换，切换时自动 `ollama stop` 旧模型 → 加载新模型 |
| 5 | **双语对照显示** | 桌面窗口同时显示原文和译文，方便对照阅读 |

### P1 — 重要功能

| # | 功能 | 说明 |
|---|------|------|
| 6 | **专注模式** | 只显示译文的精简窗口，适合阅读时最小化干扰 |
| 7 | **自动复制结果** | 翻译完成后，将译文自动复制回剪贴板，可直接粘贴使用 |
| 8 | **自动粘贴（替换选中文本）** | 自动复制后模拟 Ctrl+V，直接替换原文（划词翻译体验） |
| 9 | **智能互译** | 自动检测源语言，根据设置配对翻译。如设置英↔中，复制英语→翻译为中文，复制中文→翻译为英文 |
| 10 | **增量复制** | 将新复制的文本追加到原文末尾而非替换，适合跨页分段复制 |
| 11 | **全局快捷键** | 可自定义快捷键：翻译/切换模式/切换模型/切换语言等 |

### 4.2 模块职责

#### 4.2.1 剪贴板管理器 (ClipboardManager)

- 周期性轮询剪贴板内容（或使用操作系统 API 监听）
- 检测内容变化（比较 hash 避免重复翻译）
- 去重：同一段文本不重复触发
- 读取纯文本格式，兼容富文本（自动降级）
- **平台方案**：Python 下用 `pyperclip` + `pynput` 或 `win32clipboard`；或 Rust 下用 `arboard`

#### 4.2.2 文本净化器 (TextCleaner)

核心处理管道（可配置启用/禁用每个环节）：

```
原始剪贴板文本
    ↓
[1] 去除空白字符（BOM/零宽空格等不可见字符）
    ↓
[2] 修复连字符断词（"transla-\ntion" → "translation"）
    ↓
[3] 合并段落内换行（行末无句号的换行合并）
    ↓
[4] 保留段落分隔（双换行/缩进段落保持独立）
    ↓
[5] 中文/英文特定优化（中文不空格、数字/单位保护等）
    ↓
净化后文本 → 送入翻译
```

#### 4.2.3 翻译调度器 (TranslationDispatcher)

- 根据当前配置的源语言/目标语言构建 prompt
- 调用 Ollama API，解析流式/非流式响应
- 处理翻译超时/失败/截断的重试和回退
- 不同模型使用各自的 prompt 模板：

```python
# translategemma:4b 官方推荐格式
prompt = (
    f"<bos>You are a professional {source_lang} to {target_lang} translator. "
    f"Your goal is to accurately convey the meaning and nuances of the original "
    f"{source_lang} text while adhering to {target_lang} grammar, vocabulary, "
    f"and cultural sensitivities. Produce only the {target_lang} translation, "
    f"without any additional explanations or commentary.\n"
    f"Please translate the following {source_lang} text into {target_lang}: "
    f"{text}"
)

# hy-mt1.5:1.8b 格式（相似但更简洁）
prompt = f"Translate the following {source_lang} text to {target_lang}: {text}"
```

#### 4.2.4 Ollama API 客户端

- 封装 Ollama REST API
- 支持模型列表查询（`GET /api/tags`）
- 支持翻译请求（`POST /api/generate`）
- 支持模型卸载（`POST /api/generate` 带 `keep_alive=0`）
- 支持流式输出/非流式输出切换
- 请求超时和重试机制

#### 4.2.5 UI 层

- **对照模式**：左右分栏或上下分栏，原文+译文同步显示
- **专注模式**：独立精简窗口，仅显示译文，支持贴边隐藏
- **设置面板**：模型选择、语言设置、快捷键、文本净化选项、主题等
- **托盘图标**：常驻系统托盘，右键菜单快速操作

### 4.3 模型管理

#### 模型切换流程

```
用户选择模型 B
    ↓
[1] 检查 B 是否已下载（ollama list）
    ↓ 未下载
[2] ollama pull B（显示进度）
    ↓ 已下载或下载完成
[3] ollama stop A（卸载当前模型）
    ↓
[4] 调用 Ollama API 测试 B 是否正常
    ↓
[5] 更新配置，通知 UI 刷新
```

#### 翻译时模型生命周期

```
发送翻译请求时 → Ollama 自动加载模型到内存
                  ↓
连续翻译 → 模型保持热加载（keep_alive=-1）
          ↓
空闲超时（可配，默认 5 分钟）→ ollama stop 卸载
```

## 6. 技术选型

### 6.1 技术方案对比

| 方案 | 语言 | 优点 | 缺点 |
|------|------|------|------|
| **Python** 🥇 | Python 3.11+ | 开发快、生态丰富（pyperclip/pynput/requests/tkinter/Qt） | 分发略重、需打包 |
| Rust | Rust | 性能极致、单二进制分发 | 开发周期长、GUI 生态弱 |
| Electron | JS/TS | 跨平台、UI 漂亮 | 打包体积大、内存占用高 |
| Tauri | Rust + Web | 跨平台、体积小、UI 灵活 | 复杂功能需 Rust 侧实现 |

**推荐：Python + PyQt6 (或 tkinter)**

理由：
- 最快速验证 MVP
- 剪贴板/系统托盘/Python 生态最成熟
- 未来可编译为单 exe（PyInstaller/Nuitka）
- 与 Ollama API 交互天然友好（requests）

### 6.2 关键依赖（Python 方案）

| 用途 | 库 | 说明 |
|------|----|------|
| GUI 框架 | PyQt6 或 tkinter | 桌面窗口、托盘图标 |
| 剪贴板 | pyperclip | 跨平台剪贴板读写 |
| 剪贴板监听 | pynput 或 win32clipboard | 监听剪贴板变化 |
| HTTP 客户端 | httpx (推荐) 或 requests | 调用 Ollama API，支持流式 |
| 配置管理 | pydantic-settings | 类型安全的配置 |
| 数据持久化 | sqlite3 (内置) | 翻译历史 |
| 打包 | Nuitka 或 PyInstaller | 分发单文件 exe |
| 语言检测 | lingua-py 或 fast-langdetect | 离线语言识别 |

### 6.3 Ollama API 调用示例

```python
import httpx

def translate(text: str, model: str, source: str, target: str) -> str:
    prompt = build_prompt(text, source, target)  # 按模型构建
    payload = {
        "model": model,
        "prompt": prompt,
        "stream": False,
        "options": {
            "temperature": 0.0,     # 翻译用最低温度
            "num_predict": 2048,    # 最大生成长度
        },
        "keep_alive": -1,           # 保持模型加载
    }
    with httpx.Client(timeout=60.0) as client:
        resp = client.post("http://localhost:11434/api/generate", json=payload)
        resp.raise_for_status()
        return resp.json()["response"].strip()
```

## 8. 实现阶段（增量迭代计划）

每个阶段独立可交付，有明确的验收标准。按照依赖关系排序，但互不依赖的阶段可以并行推进。

---

### Phase 1 — 项目脚手架 + 配置管理

**工作量**：~1 次编码

**内容**：
- 创建项目目录结构（`core/`, `ui/`, `resources/`）
- `requirements.txt`: 定义基础依赖（httpx, pyperclip, pydantic-settings 等）
- `core/config.py`: 配置模型（Pydantic），支持 JSON 持久化，自动创建默认配置
- `main.py`: CLI 入口，输出版本号

**验收标准**：
```
python main.py --version  →  CopyTranslator-Ollama v0.1.0
python main.py --help     →  显示帮助信息
首次运行自动生成 ~/.copytranslate-ollama/config.json
```

### Phase 2 — Ollama API 客户端

**工作量**：~1 次编码

**内容**：

## 9. 边界与风险

| 风险 | 影响 | 缓解方案 |
|------|------|---------|
| 剪贴板监听轮询延迟 | 翻译不及时 | 轮询间隔可配 (100-500ms)；或使用 OS 剪贴板通知 API |
| 4B 模型翻译质量不足 | 长文本/专业领域效果差 | 支持自定义 prompt 模板；可扩展接入更大的模型 |
| 显存不足（同时加载多个模型） | 系统卡顿 | 强制一次只加载一个模型，切换时自动卸载旧的 |
| Ollama 服务未启动 | 无法翻译 | 启动时检测并在 UI 提示；可选自动启动 Ollama 进程 |
| 连续大量复制触发频繁翻译 | 性能浪费 | 去重（相同内容跳过）、节流（短时间多次复制合并或延迟） |
| 大文本翻译超时/截断 | 译文不完整 | 分块翻译 + 合并；或增大 `num_predict` |

## 12. 后续待确认/调研

### 已确定
- ✅ **GUI 框架**：推荐 PyQt6（生态成熟、跨平台、托盘支持好）。如追求轻量，候选 tkinter。
- ✅ **剪贴板监听**：先以轮询为主（简单可靠），后续可按需引入 Win32 剪贴板链提升响应速度。
- ✅ **模型选择**：主力 translategemma:4b，备选 hy-mt1.5:1.8b，放弃 gemma3-translator:1b。

### 待调研
1. **纯净输出测试**：`translategemma:4b` 在翻译长文本/专业文献时能否严格做到只输出译文不含额外说明？需要收集更多测试数据。
2. **离线语言检测**：对比 `lingua-py` (高精度但慢) vs `fast-langdetect` (快但可能精度差)，选最优方案。
3. **文本净化效果验证**：收集不同 PDF 的真实排版样本，验证净化管道的鲁棒性。
4. **自动粘贴实现**：在 Win10/Win11 下模拟 Ctrl+V 的可靠方案（`pyautogui`/`keyboard`/`SendKeys`）。
5. **全局快捷键库选择**：`keyboard`（需要管理员权限 on Windows）vs `pynput` vs `system_hotkey`。
6. **打包方案**：Nuitka vs PyInstaller，对比 exe 体积、启动速度、兼容性。
7. **Ollama 服务检测**：如何在启动时检测 Ollama 是否运行，如未运行是否自动启动。
